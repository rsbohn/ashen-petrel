# ADXA/ADBX Family Comparison (SIMH vs Ashen)

Setup used for both:
- Memory at `0200`:
  - `021020` (LDI 0020)
  - `021010` (LDI 0010)
  - `021430` (LDXI 0030)
  - `003500` (XAX, NOP)
  - `003700` (ADXA, NOP)
  - `007300` (INCB, NOP)
  - `007400` (DECB, NOP)
  - `007500` (XBX, NOP)
  - `007600` (ADBX, NOP)
  - `007700` (ADXB, NOP)
- Initial state after setup (octal): `X=000030`, `A=000010`, `B=000020`.

Results (state after each instruction, octal):

| Instruction | SIMH (X, A, B) | Ashen (X, A, B) | Match |
| --- | --- | --- | --- |
| XAX | `000010, 000030, 000020` | `000010, 000030, 000020` | Yes |
| ADXA | `000010, 000040, 000020` | `000010, 000040, 000020` | Yes |
| INCB | `000010, 000040, 000021` | `000010, 000040, 000021` | Yes |
| DECB | `000010, 000040, 000020` | `000010, 000040, 000020` | Yes |
| XBX | `000020, 000040, 000010` | `000020, 000040, 000010` | Yes |
| ADBX | `000030, 000040, 000010` | `000030, 000040, 000010` | Yes |
| ADXB | `000030, 000040, 000040` | `000030, 000040, 000040` | Yes |
